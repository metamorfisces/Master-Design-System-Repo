# Color Style System

**Overview**

This color system is designed to be flexible across multiple industries and website templates while maintaining a professional, modern, and trustworthy visual identity.

The system uses semantic color roles rather than component-specific assignments, allowing templates to apply colors based on context while maintaining consistency.

# Design Principles

 1.    
    Use semantic color roles rather than component-specific assignments.
    
 2.    
    Prioritize hierarchy and readability over decoration.
    
 3.    
    Maintain consistent contrast ratios across all templates.
    
 4.    
    Allow individual templates to interpret color usage while preserving the system’s structure.
    
 5.    
    Use Accent colors for emphasis, Background colors for structure, Text colors for hierarchy, and Stroke colors for separation.
 6. Accessibility & Contrast Standards - Accessibility should be considered during all design decisions, particularly when selecting text, background, CTA, and interactive element combinations.  


**Minimum Contrast Requirements**
  
Following WCAG AA guidelines:  
  
- **Regular Text:** Minimum contrast ratio of **4.5:1**  
- **Large Text:** Minimum contrast ratio of **3:1**  
- Large text is defined as:  
- 18pt (24px) or larger  
- 14pt (18.66px) or larger when bold  
  
**Best Practices**

- Validate contrast ratios before introducing new color combinations.  
- Do not rely solely on color to communicate meaning or status.  
- Ensure CTA buttons maintain sufficient contrast in all states (default, hover, active, disabled).  
- Verify text readability on both light and dark backgrounds.  
- Prioritize readability and accessibility over aesthetic preference when conflicts arise.  
  

## Goal

  
The design system should strive to meet or exceed WCAG AA accessibility standards across all templates and components while maintaining a professional visual appearance.
